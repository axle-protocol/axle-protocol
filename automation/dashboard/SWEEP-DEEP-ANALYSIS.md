## 2. 제품 기능 상세 분석

### 2-1. 핵심 기능 목록

스윕의 기능을 카테고리별로 상세 정리하면 다음과 같다.

#### A. 주문 수집 (Order Collection)

| 세부 기능 | 상세 설명 |
|-----------|----------|
| **멀티채널 주문 수집** | 네이버 스마트스토어, 쿠팡(Coupang Wing), 11번가, ESM(옥션/지마켓), 카카오톡 스토어 등 주요 마켓에서 주문을 실시간으로 수집 |
| **글로벌 마켓 지원** | 알리익스프레스, 테무(Temu) 등 해외 플랫폼 주문 관리 지원 |
| **주문 통합** | 여러 판매처의 주문 파일을 하나의 파일로 자동 통합 |
| **실시간 동기화** | API 기반 실시간 주문 데이터 동기화 |
| **주문 상태 구분** | 알리익스프레스 등의 모호한 "배송 대기중" 상태를 더 명확한 상태로 세분화하여 표시 |

#### B. 발주 자동화 (Purchase Order Automation)

| 세부 기능 | 상세 설명 |
|-----------|----------|
| **공급사별 발주서 자동 생성** | 수집한 주문 데이터를 공급사별로 자동 분류하고, 각 공급사의 엑셀 양식에 맞춰 발주서 파일 자동 생성 |
| **발주서 이메일 자동 전송** | 생성된 발주서 파일을 공급사에게 이메일로 자동 전송 |
| **엑셀 포맷 변환** | 판매처마다 다른 주문서 형식을 공급사가 원하는 형식으로 자동 변환 |
| **복수 주문 분리** | 한 고객이 여러 상품을 한번에 주문한 경우 개별 품목으로 자동 분리하여 발주 |

#### C. 배송/송장 관리 (Shipping & Tracking)

| 세부 기능 | 상세 설명 |
|-----------|----------|
| **송장번호 자동 등록** | 공급사로부터 받은 송장번호를 해당 마켓플레이스에 자동 등록 |
| **묶음배송 자동 분류** | 같은 고객의 여러 주문을 하나의 배송으로 묶는 합배송 자동 처리 |
| **합배송 처리** | 하나의 주문 내 여러 상품을 합쳐서 발송하는 처리 자동화 |
| **나눔배송 처리** | 하나의 주문을 여러 배송건으로 나누어 별도 송장 부여 자동 처리 |
| **송장 분할** | 알리익스프레스 등에서 부분 발송 시 트래킹 번호 자동 분할 |

#### D. 취소/반품/교환 (CS/Claims)

| 세부 기능 | 상세 설명 |
|-----------|----------|
| **주문 취소 자동 처리** | 여러 플랫폼의 취소 요청을 통합 관리하여 균일하게 처리 |
| **취소 승인/거부** | 실시간 취소 요청 확인 및 승인/거부 처리 |
| **교환/반품 관리** | 교환, 반품, 취소 등 CS 업무 통합 처리 지원 |

#### E. 정산 관리 (Settlement)

| 세부 기능 | 상세 설명 |
|-----------|----------|
| **자동 정산 데이터 생성** | 판매처별/공급사별 판매량과 매출 데이터를 자동으로 생성 |
| **일자별/거래처별 정산** | 다양한 조건별로 정산 데이터를 자동 정리 |

## 3. 가격 정책

### 공개된 정보

스윕의 정확한 요금은 **비공개**이다. 다만 수집된 정보를 종합하면:

| 항목 | 내용 |
|------|------|
| **무료 체험** | 21일(3주) 모든 기능 무료 사용 |
| **가격 포지셔닝** | "월등히 저렴" 표방, 사방넷/플레이오토 대비 상당히 낮은 가격대 |
| **과금 방식** | 월 구독형 (정확한 금액 비공개) |
| **ROI 메시지** | "월 구독비보다 배송 실수 2-3건 방지로 절감되는 비용이 더 크다" |
| **인력 대체 효과** | 인건비 대비 2배 이상 비용 효율적이라고 주장 |

### 가격 추정

- 사방넷 일반: 월 15만~52만원
- 플레이오토: 월 16만~60만원
- 셀로: 월 38,500원
- **스윕 추정: 월 3만~10만원 범위** (경쟁사 대비 "월등히 저렴"이면서 수익화 가능한 수준)

### 4-2. 콘텐츠 마케팅 전략

스윕은 **블로그 콘텐츠 마케팅에 상당한 투자**를 하고 있다.

**블로그 콘텐츠 카테고리:**

| 카테고리 | 대표 글 | 목적 |
|---------|--------|------|
| **셀러 가이드** | 테무 입점 가이드, 알리익스프레스 셀러 가이드, 토스쇼핑 입점 | SEO + 잠재 고객 유입 |
| **솔루션 비교** | 사방넷 vs 플레이오토 vs 스윕 완벽 비교 | 직접 전환 유도 |
| **사용자 후기** | 주왕산가든 도입후기, 위플 도입후기 | 사회적 증거 (Social Proof) |
| **실무 가이드** | 묶음배송/합배송 자동화, 재고관리 솔루션 TOP5 | 전문성 확보 + SEO |
| **마켓 트렌드** | 도매매 vs 도매꾹, 상품명 전략 | 셀러 커뮤니티 내 브랜드 인지도 |
| **수수료 계산기** | 알리익스프레스 수수료 무료 엑셀 계산기 | 리드 제너레이션 |

**분석:** 스윕은 8명이라는 소규모 팀에도 불구하고 상당한 양의 교육 콘텐츠를 생산하고 있다. 특히 알리익스프레스, 테무 등 **신규 플랫폼 셀러를 타겟으로 한 가이드 콘텐츠**가 많아, 이 채널을 통해 신규 셀러를 유입시키는 전략이 명확하다.

### 4-3. 고객 사례 (Case Studies)

#### 사례 1: 주왕산가든 (프리미엄 한우 레스토랑 → 이커머스)

| 항목 | 상세 |
|------|------|
| **업종** | 30년 전통 프리미엄 한우 식당 → 온라인 판매 |
| **규모** | 네이버 스마트스토어 월 매출 3~4억원, 프리미엄 셀러 |
| **온라인 vs 오프라인** | 온라인 매출이 오프라인의 2배 이상 |
| **도입 전** | 매일 3시간+ 데이터 변환 및 주문 처리 |
| **도입 후** | 5분으로 단축 (97% 시간 절감) |
| **핵심 효과** | 직원 스트레스 해소, 주문 실수 제로화, 비용 절감 |

#### 사례 2: 위플 (더신선 / 일류농사꾼 브랜드)

| 항목 | 상세 |
|------|------|
| **업종** | 신선식품 온라인 판매 (2개 브랜드 운영) |
| **규모** | 직원 30명, 연매출 150억원 (2024), 2025년 250억 목표 |
| **판매 채널** | 네이버 스마트스토어, 카카오 메이커스, 알리익스프레스 등 멀티채널 |
| **도입 전 문제** | 30개+ 공급사의 각기 다른 발주서 양식 관리, 플랫폼별 다른 주문 포맷 |
| **핵심 효과** | 취소 처리 자동화 (주말 1,000건+ 취소를 30~40분에서 자동으로), 신속한 주문 수집 |

### 5-2. 자동화 파이프라인 비교

```
주문 수집 → 공급사 배분 → 발주서 생성 → 공급사 확인 → 송장 수집 → 발송 처리 → CS 대응

스윕:    [자동(API)] → [자동]     → [자동(엑셀)] → [수동(이메일)] → [반자동]  → [자동]     → [수동]
우리현재: [수동(엑셀)] → [자동]    → [불필요]     → [자동(포탈)]  → [자동(포탈)]→ [반자동]   → [없음]
우리계획: [자동(API)] → [자동(AI)] → [불필요]     → [자동(포탈)]  → [자동(포탈)]→ [자동(API)]→ [자동(AI)]
```

**핵심 인사이트:**
- **스윕의 접근:** 기존 엑셀 기반 워크플로우를 자동화 (엑셀 → 엑셀을 자동으로)
- **우리의 접근:** 엑셀 자체를 없애고 벤더 포탈로 근본적 전환 (양방향 디지털 시스템)
- **근본적 차이:** 스윕은 "더 빠른 엑셀 처리", 우리는 "엑셀 없는 세상"

## 6. 스윕의 강점 분석 (우리가 배워야 할 점)

### 6-1. 스윕이 우리보다 잘하는 것

| 5 | **글로벌 마켓** | 알리익스프레스, 테무 주소변환/상품명 변환 등 특화 기능 | 미지원 | **중간** |

### 6-2. 가장 위협적인 시나리오

1. **스윕이 벤더 포탈을 만들면?**
   - 투자금 15억+을 보유하고 있어 개발 리소스 충분
   - 하지만 현재까지 엑셀 기반 접근법을 고수 중 (2024.1 설립 ~ 2026.2 현재까지 2년간 미구현)
   - 엑셀 기반 워크플로우는 스윕의 핵심 아이덴티티이므로 전환 어려울 수 있음

2. **스윕이 AI 기능을 추가하면?**
   - 현재 팀 규모(8명)와 기술 스택으로는 단기간 내 AI 구현 어려움
   - 그러나 LLM API 기반 간단한 기능(주소 변환, 상품명 번역)은 이미 적용 중일 수 있음
   - AI CS, AI 에이전트 등 깊은 수준의 AI는 시간 소요

3. **스윕의 네트워크 효과**
   - 현재 많은 공급사가 이미 스윕 형식의 엑셀 발주를 수용하고 있을 것
   - 이 관성이 공급사의 벤더 포탈 전환을 방해할 수 있음

## 7. 스윕의 약점 분석 (우리의 기회)

### 7-1. 스윕이 못하는 것/약한 것

| # | 약점 | 상세 | 우리의 기회 |
|---|------|------|-----------|
| 1 | **공급사 측 시스템 없음** | 공급사와의 소통은 여전히 이메일/카톡/전화. 발주서를 "보내줄 뿐" 공급사의 확인/응답/송장 입력을 시스템화하지 못함 | **벤더 포탈로 근본 해결** |

| 3 | **CS 자동화 없음** | 고객 문의는 셀러가 직접 대응해야 함 | **LLM 기반 AI CS** |

### 7-2. 구조적 약점

1. **엑셀 의존적 아키텍처**
   - 스윕의 핵심 가치가 "엑셀을 잘 처리해줌"인데, 이는 레거시 워크플로우에 최적화된 것
   - 장기적으로 "엑셀 없는 워크플로우"가 더 효율적
   - 공급사가 벤더 포탈에 익숙해지면, 엑셀 발주서가 오히려 비효율로 느껴질 것

2. **단방향 시스템**
   - 셀러만 사용하는 도구. 공급사는 시스템 밖에 있음
   - 양방향 네트워크 효과를 만들 수 없는 구조
   - 셀러 이탈 시 전환 비용이 낮음 (공급사 lock-in 없음)

3. **크롬 확장프로그램 의존**
   - 일부 기능이 크롬 확장프로그램에 의존
   - 브라우저 업데이트, 정책 변경 등에 취약

## 11. 종합 결론

### 스윕은 어떤 경쟁자인가?

스윕은 **"위탁판매 발주 자동화"라는 동일한 문제를 풀고 있는 직접 경쟁자**다. 8명의 팀과 15억원의 자금, 빠른 성장세를 갖추고 있으며, 이미 유의미한 고객 기반을 확보하고 있다.

그러나 스윕의 접근법은 **"기존 엑셀 워크플로우를 더 빠르게 처리"**하는 것이고, 우리의 접근법은 **"엑셀 워크플로우 자체를 없애고 AI로 대체"**하는 것이다. 이 근본적 차이가 우리의 핵심 경쟁 무기다.

### 핵심 승부처

| 승부처 | 스윕 | 우리 | 판단 |
|--------|------|------|------|
| **공급사 관리** | 이메일/엑셀 단방향 | 벤더 포탈 양방향 | **우리 우위** |
| **AI 자동화** | 없음 | AI 에이전트 + AI CS | **우리 우위** |
