## 2. 시장 Pain Points (핵심 불만 사항)

### 2-1. 위탁판매 송장 관리의 고통
- **핵심 문제:** 오픈마켓 주문이 들어올 때마다 도매 사이트에서 하나씩 발주하고, 운송장을 복붙하는 과정이 반복됨
- **실제 셀러 목소리:** "예전에는 하루 절반이 송장 처리로 날아갔다"
- **공급사별 다른 양식:** 각 공급사마다 엑셀 양식이 달라 발주서를 공급사별로 분리 작성해야 함
- **공급사와의 소통:** 대부분 카카오톡이나 전화로 발주/송장 확인 → 체계적 관리 불가

### 2-2. 기존 솔루션의 소규모 셀러 부적합성
- **사방넷/플레이오토/이지어드민:** 기능이 너무 많고 복잡, 초기 세팅 자체가 진입장벽
- **비용 문제:** 모든 기능 번들 가격을 지불해야 하지만 실제 사용하는 기능은 일부
- **위탁판매 특화 부재:** 대부분 "자체 재고 보유 셀러" 기준으로 설계되어 있음

### 2-3. CS(고객 서비스) 자동화의 부재
- **셀러의 고통:** "불만을 가지고 CS를 요청하는 구매자와 통화하다 보면 정작 해야 될 중요한 일을 놓치게 됨"
- **톡톡 자동응답 한계:** 네이버 톡톡의 기본 자동응답은 단순 키워드 매칭 수준, AI 기반 맞춤 응답 불가
- **반복 문의 처리:** 배송 추적, 교환/환불 안내 등 반복 문의가 CS의 대부분을 차지하지만 자동화 도구 부재
- **CS와 주문 데이터 단절:** CS 문의 내용과 주문 정보가 연결되지 않아 매번 주문 확인 후 답변

### 2-4. 네이버 커머스 API의 제약
- **주문 조회 24시간 제한:** 조회 기준 범위가 최대 24시간으로만 설정 가능, 장기간 데이터 수집 시 매일 반복 호출 필요
- **Rate Limit:** 내스토어 애플리케이션 기준 초당 2회(2/s), HTTP 429 반환
- **전체 주문 조회 불가:** 특정 기간 전체 주문을 한 번에 조회하는 기능 미제공
- **IP 등록 제한:** 최대 3개 IP만 등록 가능
- **주기적 인증 필요:** 애플리케이션별 인증 기한이 있으며 만료 시 휴면 처리

## 3. CS 자동화 현황 분석

### 3-1. 네이버 톡톡 챗봇 API
- **기본 구조:** Chat Bot API 기반, Webhook 방식으로 메시지 수신/발신
- **메시지 전송:** 동기식(5초 이내 응답) / 비동기식(send API로 직접 전송)
- **지원 형식:** textContent, imageContent, compositeContent (1개만 선택 가능)
- **주요 제약:**
  - 네이버 측에서 상담 종료 API 미제공 → 외부 서비스에서 답변 후에도 톡톡에서는 미답변으로 남음
  - 푸시 알림 사용 제한
  - TLS 필수, 정식 인증서만 사용 가능
  - 종료된 상담창에서 대화 이어가기 불가
- **출처:** [GitHub navertalk/chatbot-api](https://github.com/navertalk/chatbot-api)

### 3-3. AI 기반 CS 자동화의 현재 수준
- 카페24 PRO가 "묻기도 전에 해결한다"는 컨셉으로 선제적 FAQ 자동 보강 시작 (2025.12)
- 대부분의 솔루션은 **규칙 기반(Rule-based)** 자동응답 수준
- **LLM 기반 맞춤 CS 자동응답**을 제공하는 스마트스토어 전용 솔루션은 **아직 없음**
- 위탁판매 특화 CS (배송조회, 공급사 확인, 교환/반품 프로세스 자동화)를 통합 제공하는 솔루션은 **전무**

## 4. 네이버 커머스 API 역량 분석

### 4-1. API 기능 범위
| 영역 | 지원 여부 | 세부 내용 |
|------|----------|----------|
| **상품 관리** | O | 상품 등록/수정/삭제, 카테고리 관리 |
| **주문 조회** | O (제한적) | 신규 주문, 변경 주문 조회 (최대 24시간 범위) |
| **발송 처리** | O | 주문 상태 변경, 송장번호 입력/업데이트 |
| **클레임 관리** | O | 반품/교환/취소 처리 |
| **CS 문의** | O | 판매자 문의/구매후기/CS문의 관리 |
| **재고 연동** | O | SKU 기준 재고 동기화 |

### 4-2. 인증 방식
- **OAuth 2.0** (ClientCredentials flow)
- Token URL: `https://api.commerce.naver.com/external/v1/oauth2/token`
- 내스토어 애플리케이션 등록 필요 (스토어당 최대 1개)
- API 호출 IP 등록 필수 (최대 3개)
- 주기적 인증 갱신 필요 (미갱신 시 휴면 처리)

### 4-3. Rate Limit
- 내스토어 애플리케이션: **초당 2회 (2/s)**
- 솔루션 애플리케이션: 더 높은 호출량 (2025 하반기 정식 출시 예정)
- 초과 시 HTTP 429 반환
- 토큰 버킷 알고리즘 적용

### 4-4. 핵심 제약사항
1. 전체 주문 한번에 조회 불가 (24시간 단위로 분할 호출 필요)
2. Rate Limit 사용자 임의 제어 불가
3. IP 등록 3개 제한
4. 톡톡 챗봇 API와 커머스 API는 별개 (상담-주문 연동은 직접 구현 필요)
5. 솔루션 애플리케이션 등록 시 네이버 심사 필요

### 4-5. 최근 변화 (2025-2026)
- 기존 SOAP API 2023.11.30 종료 → REST 기반 커머스 API 전환 완료

- 커머스솔루션마켓 내 솔루션 유료화 시작 (월 1,000원~25,000원)

## 5. 시장 갭 & 차별화 기회

### 5-1. TOP 미충족 니즈

#### (1) "셀러-공급사 간 소통/협업" 자동화 시스템 부재 ★★★★★
- **현재:** 카카오톡/전화/이메일로 발주, 엑셀로 송장 주고받음
- **문제:** 체계적 관리 불가, 누락 발생, 히스토리 추적 어려움
- **기회:** 공급사 전용 포탈(벤더 포탈)은 현재 시장에 **전무**
- **차별화:** 공급사가 직접 주문 확인하고 송장번호 입력하는 웹 포탈 → 셀러가 확인 → 자동 업로드

#### (2) 위탁판매 전용 "풀 자동화" 파이프라인 부재 ★★★★★
- **현재:** 주문수집 → (수동) 발주 → (수동) 송장 확인 → (수동) 송장 업로드
- **문제:** 각 단계마다 사람 개입 필요, 스윕이 일부 해결하지만 공급사 측 시스템 없음
- **기회:** 주문수집 → 자동 배분 → 공급사 확인/송장입력 → 자동 수집 → 자동 업로드의 End-to-End 자동화

#### (3) AI 기반 CS 자동응답 (네이버 톡톡) ★★★★☆
- **현재:** 규칙 기반 자동응답만 존재, LLM 기반 맞춤 응답 없음
- **문제:** 반복 CS 문의가 셀러 시간의 상당 부분 차지
- **기회:** 주문 데이터와 연동된 AI CS (배송 어디쯤 왔나요? → 실시간 배송 조회 후 자동 응답)

#### (4) 소규모 셀러를 위한 "적정 가격 + 적정 기능" ★★★★☆
- **현재:** 무료(기능 부족) vs 고가(기능 과잉)의 양극화
- **문제:** 1인 셀러에게 필요한 핵심 기능만 합리적 가격으로 제공하는 솔루션 부재
- **기회:** 위탁판매 핵심 워크플로우만 자동화하는 월 2-5만원대 SaaS

### 5-2. 기존 솔루션이 놓치고 있는 것

| 기존 솔루션의 초점 | 누락된 영역 (우리의 기회) |
|-------------------|------------------------|
| 상품 등록/관리 | 공급사 관리/소통 |
| 주문 수집 | 공급사 자동 배분 |
| 송장 일괄 전송 | 공급사로부터 송장 자동 수집 |
| 재고 관리 | 위탁판매용 가상 재고 관리 |
| 규칙 기반 CS | AI 기반 맥락 인식 CS |
| PC 설치형 | 모바일 퍼스트/웹 기반 |
| 셀러만 사용 | 셀러 + 공급사 양방향 플랫폼 |

### 5-3. 제안하는 차별화 전략

#### 핵심 차별화: "벤더 포탈" (공급사 전용 시스템)
- 현재 시장에 **공급사 측에서 직접 주문을 확인하고 송장을 입력하는 시스템은 존재하지 않음**
- 이것이 가장 강력한 경쟁 우위
- 구현: 공급사에게 간단한 웹 링크 제공 → 로그인 → 배분된 주문 확인 → 송장번호 입력 → 셀러 측 자동 반영

#### 보조 차별화: AI CS 자동응답
- 네이버 톡톡 챗봇 API + LLM 연동
- 주문 데이터 기반 맥락 인식 응답
- "배송 어디까지 왔나요?" → 자동으로 송장번호 조회 → 배송 상태 답변
- "교환하고 싶어요" → 자동으로 교환 가이드 + 반품 접수

#### 가격 차별화: 1인 셀러 맞춤 가격
- 월 2-5만원대 (사방넷 미니보다 기능 특화, 사방넷 일반보다 저렴)
- 주문 건수 기반 종량제 옵션

## 8. 리스크 & 주의사항

1. **스윕이 직접 경쟁자:** 2024년 10억 투자 유치, 위탁판매 발주 자동화 특화 → 빠르게 기능 확장 가능
2. **네이버 API 제약:** Rate Limit(2/s), 24시간 조회 제한 → 다수 셀러 서비스 시 솔루션 애플리케이션 등록 필요
3. **공급사 온보딩:** 벤더 포탈의 핵심은 공급사가 실제로 사용하느냐 → 공급사의 IT 리터러시, 기존 워크플로우 변경 저항
4. **네이버 커머스솔루션마켓:** 네이버 생태계 내 배포 채널로 활용 가능하지만, 네이버의 자체 솔루션과 경쟁 가능성
5. **사방넷 미니 가격 인하:** 월 3,000원으로 초보 셀러 시장 진입 → 가격 경쟁 주의

## 9. 전략적 제언

### 단기 (MVP)
1. 스마트스토어 주문 수집 → 공급사 자동 배분 → 벤더 포탈 → 송장 자동 업로드
2. 네이버 커머스 API + 간단한 웹 기반 벤더 포탈
3. 타겟: 공급사 3-10개, 일 주문 10-100건 수준의 1인 위탁판매 셀러

### 중기 (확장)
1. AI CS 자동응답 (네이버 톡톡 챗봇 API + LLM)
2. 다중 마켓 지원 (쿠팡, 11번가 등)
3. 정산/매출 분석 대시보드

### 장기 (플랫폼)
1. 셀러-공급사 매칭 플랫폼으로 확장
2. 공급사 네트워크 효과 활용
3. SaaS + 마켓플레이스 하이브리드 모델
