## 1. 한국 장소/맛집 데이터를 가져올 수 있는 무료 API 목록

### ① 카카오 로컬 API (Kakao Local)
- **URL**: `https://dapi.kakao.com/v2/local/...`
- **문서**: https://developers.kakao.com/docs/latest/ko/local/dev-guide
- **주요 기능**:
  - 키워드로 장소 검색 (`/v2/local/search/keyword`)
  - 카테고리로 장소 검색 (`/v2/local/search/category`)
  - 주소 ↔ 좌표 변환
  - 좌표 → 행정구역 변환
- **제공 데이터**: 장소명, 카테고리, 주소, 도로명주소, 전화번호, 좌표(위경도), 장소 URL
- **인증**: REST API 키 (카카오 개발자 앱 등록)

### ② 네이버 검색 API - 지역 (Naver Search Local)
- **URL**: `https://openapi.naver.com/v1/search/local.json`
- **문서**: https://developers.naver.com/docs/serviceapi/search/local/local.md
- **주요 기능**:
  - 네이버 지역 서비스 등록 업체/기관 검색
  - 정확도순, 리뷰수순 정렬
- **제공 데이터**: 업체명, 카테고리, 설명, 지번/도로명 주소, 좌표, 상세 URL(네이버 플레이스 링크)
- **인증**: Client ID + Client Secret (네이버 개발자 센터)

### ③ 네이버 검색 API - 블로그/이미지
- **블로그**: `https://openapi.naver.com/v1/search/blog.json` — 맛집 블로그 리뷰 검색
- **이미지**: `https://openapi.naver.com/v1/search/image` — 장소 관련 이미지 검색
- 블로그 리뷰를 간접적으로 수집하여 장소 정보 보완 가능

### ④ 한국관광공사 TourAPI (공공데이터 포털)
- **데이터셋**: 한국관광공사_국문 관광정보 서비스_GW
- **공공데이터 포털**: https://www.data.go.kr/data/15101578/openapi.do
- **TourAPI 포털**: https://api.visitkorea.or.kr
- **주요 기능 (15종 API)**:
  - 지역기반 관광정보 조회
  - 위치기반 관광정보 조회
  - 키워드 검색
  - 행사정보 조회
  - 숙박정보 조회
  - 공통정보 / 소개정보 / 반복정보
  - **이미지정보 조회** ✅
  - 반려동물 동반여행 정보
- **제공 데이터**: 약 26만 건, 관광지/음식점/숙박/쇼핑/문화시설/축제 등
- **사진**: 공공누리 1유형/3유형 이미지 제공 (상업적 이용 가능, 단 명예훼손/CI·BI 사용 금지)
- **인증**: 공공데이터 포털 API 키 (자동승인)

### ⑤ 공공데이터 포털 - 지자체 음식점 데이터
- **URL**: https://www.data.go.kr
- **주요 데이터셋 예시**:
  - 경기도 일반음식점/휴게음식점 현황
  - 식품의약품안전처 식품모범음식점
  - 대구광역시 먹거리골목별 음식점
  - 대전광역시 관내 음식점 정보
  - 각 시/군/구별 음식점 인허가 정보
- **제공 데이터**: 상호명, 주소, 업종, 메뉴, 영업시간, 좌석수, 주차장 등
- **특징**: 지자체별 파편화, 데이터 형식 비통일, 리뷰/별점 없음

### ⑥ Google Places API
- **URL**: https://maps.googleapis.com/maps/api/place/...
- **문서**: https://developers.google.com/maps/documentation/places/web-service
- **주요 기능**:
  - Text Search / Nearby Search
  - Place Details (리뷰, 별점, 사진 포함)
  - Place Photos
  - Autocomplete
- **제공 데이터**: 장소명, 주소, 좌표, **별점, 리뷰 텍스트, 사진** ✅
- **인증**: Google Cloud API 키
- **비용**: 매월 $200 무료 크레딧 (약 28만원), 초과 시 유료

### ⑦ 소상공인시장진흥공단 상권정보 API
- **공공데이터 포털**: https://www.data.go.kr
- **주요 기능**: 상권/업종 분석, 상권 내 업소 목록
- **제공 데이터**: 상호명, 업종분류, 주소, 좌표

### ⑧ Foursquare Places API
- **URL**: https://developer.foursquare.com/
- **주요 기능**: 장소 검색, 상세정보, 사진, 팁(리뷰)
- **제공 데이터**: 장소명, 카테고리, 좌표, 사진, 짧은 팁(리뷰)
- **무료 티어**: 월 1,000회 호출

## 2. 각 API의 제한사항 비교

| API | 일일 호출 한도 | 리뷰 데이터 | 별점 | 사진 | 비용 |
|-----|---------------|------------|------|------|------|
| **카카오 로컬** | 키워드검색 10만건/일, 카테고리검색 10만건/일 (월 300만건 무료) | ❌ 없음 | ❌ 없음 | ❌ 없음 (장소URL만) | 무료쿼터 초과시 2원/건 |
| **네이버 지역검색** | 25,000건/일 | ❌ 없음 (리뷰수 정렬만 가능) | ❌ 없음 | ❌ 없음 | 무료 |
| **네이버 블로그검색** | 25,000건/일 | ⚠️ 간접 (블로그 글) | ❌ 없음 | ⚠️ 간접 (블로그 썸네일) | 무료 |
| **네이버 이미지검색** | 25,000건/일 | ❌ 없음 | ❌ 없음 | ✅ 이미지 URL | 무료 |
| **TourAPI (한국관광공사)** | 개발: 1,000건/일, 운영: 증설 가능 | ❌ 없음 | ❌ 없음 | ✅ 공식 사진 | 무료 |
| **공공데이터 (지자체)** | API별 상이 (보통 1,000건/일) | ❌ 없음 | ❌ 없음 | ❌ 대부분 없음 | 무료 |
| **Google Places** | 무제한 (비용 기반) | ✅ 리뷰 5개 (무료), 전체는 유료 | ✅ 별점 | ✅ 사진 URL | $200/월 무료크레딧 |
| **Foursquare** | 1,000건/월 (무료) | ⚠️ Tips (짧은 리뷰) | ✅ 별점 | ✅ 사진 | 무료 티어 제한적 |

### 핵심 포인트
> **카카오/네이버 공식 API는 리뷰·별점·사진을 제공하지 않는다.**
> 이것이 데이트코스 서비스 구축의 가장 큰 허들이다.

## 3. 네이버 플레이스 / 카카오맵에서 리뷰+별점+사진을 합법적으로 가져오는 방법

### ❌ 직접 크롤링/스크래핑 — 법적 리스크 높음
- **네이버**: robots.txt로 크롤링 금지, 이용약관 위반. 2021년 대법원 판례에서 크롤링 관련 업무방해 인정
- **카카오**: 카카오맵 이용약관 상 무단 수집 금지
- ⚠️ 대규모 자동 수집은 **컴퓨터 등 장애 업무방해죄** (형법 314조 2항) 적용 가능

### ✅ 합법적 대안들

#### 방법 1: Google Places API 활용 (가장 현실적)
- Google은 API를 통해 **공식적으로 리뷰, 별점, 사진**을 제공
- 한국 장소 데이터도 상당히 풍부
- 무료 크레딧 $200/월 내에서 시작 가능
- **단점**: 네이버/카카오 대비 한국 로컬 데이터 커버리지가 낮을 수 있음

#### 방법 2: 네이버 블로그 검색 API + AI 분석
- 네이버 블로그 검색 API로 "[장소명] 후기/리뷰" 검색
- AI(LLM)로 블로그 글에서 평점, 감성, 핵심 의견 추출
- **장점**: 합법적, 풍부한 한국어 리뷰 데이터
- **단점**: 정형화되지 않음, 추가 처리 비용

#### 방법 3: 사용자 생성 콘텐츠 (UGC) 직접 구축
- 자체 리뷰 시스템 구축
- 초기 데이터는 제휴/이벤트로 확보
- 인스타그램 해시태그 기반 사진 수집 (Instagram Basic Display API → 사용자 동의 기반)
- **장점**: 완전한 소유권, 차별화 가능
- **단점**: 초기 데이터 부족 (Cold Start)

#### 방법 4: 파트너십/제휴
- 네이버, 카카오와 **공식 데이터 제휴** (사업 제안서 기반)
  - 네이버 비즈니스 플랫폼 파트너 프로그램
  - 카카오 비즈니스 제휴 문의
- 요식업 관련 플랫폼과 제휴 (배달의민족, 요기요, 캐치테이블, 다이닝코드 등)
- **현실**: 초기 스타트업은 제휴 성사가 어려움

#### 방법 5: 카카오맵/네이버 플레이스 URL 링크 + 임베드
- API로 장소 기본정보(이름, 주소, 좌표)를 가져오고
- 리뷰/별점/사진은 **해당 플레이스 페이지로 링크**하여 사용자가 직접 확인
- 카카오맵 장소 URL, 네이버 플레이스 URL을 앱 내 WebView로 표시
- **장점**: 완전히 합법, 구현 간단
- **단점**: UX 단절, 자체 데이터 축적 불가

#### 🎯 추천 전략 (단계적)
```
Phase 1: 카카오 로컬 API + TourAPI (기본 장소 데이터)
         + Google Places API (리뷰/별점/사진)
         + 네이버 블로그 API (보충 리뷰)
         → AI로 통합/분석

Phase 2: 자체 UGC 리뷰 시스템 구축
         + 사용자 인센티브 프로그램

Phase 3: 플랫폼 성장 후 네이버/카카오 공식 제휴 추진
```

## 4. 비슷한 서비스의 데이터 수집 방법

### 데이트팝 (DatePop)
- **방식**: 자체 에디터 + 파트너십 기반
- 자체 MD(머천다이저)가 직접 장소를 방문하여 큐레이션
- 음식점/카페/놀거리와 직접 제휴 (입점 계약)
- 제휴처에서 장소 정보, 사진, 할인 등을 직접 제공받음
- 사용자 리뷰는 자체 앱 내 UGC
- **특징**: 데이터 품질 높으나 커버리지 제한적, 수익은 제휴 수수료/광고

### 마이리얼트립 (MyRealTrip)
- **방식**: 가이드/호스트 등록 + 공공데이터 + API 조합
- 여행 상품은 가이드/호스트가 직접 등록 (Marketplace 모델)
- 장소 데이터는 Google Maps API + 한국관광공사 TourAPI 활용
- 사용자 리뷰/평점은 자체 UGC 시스템
- 사진은 호스트 제공 + 사용자 업로드
- **특징**: 플랫폼 규모로 데이터 선순환

### 다이닝코드 (DiningCode)
- **방식**: 블로그/SNS 리뷰 빅데이터 분석
- 네이버 블로그, 인스타그램 등의 **공개 게시물**을 분석하여 맛집 순위 생성
- 자체 알고리즘으로 리뷰 신뢰도/감성 분석
- ⚠️ 크롤링 관련 법적 이슈 여러 차례 제기됨
- **특징**: 데이터 커버리지 넓으나 법적 리스크 존재

### 망고플레이트 (MangoPlate)
- **방식**: 자체 UGC 중심
- 사용자 리뷰/별점/사진을 핵심 자산으로 구축
- 초기에 헤비 리뷰어를 초대하여 콘텐츠 확보 (Cold Start 해결)
- 식당과 제휴하여 예약 서비스 연계
- **특징**: UGC 품질 관리 (리뷰어 등급 시스템)

### 캐치테이블 (CatchTable) / 포잉 (Poing)
- **방식**: B2B 레스토랑 예약 플랫폼
- 식당이 직접 정보(메뉴, 사진, 가격)를 등록
- 예약 후 방문 확인된 사용자만 리뷰 작성 가능
- **특징**: 신뢰도 높은 리뷰, 식당 자체 제공 데이터

### 여기어때 / 야놀자
- **방식**: 숙박 플랫폼 → 여가/데이트 확장
- B2B 제휴: 펜션, 호텔, 레저시설과 직접 계약
- 업체가 사진/상세정보 직접 등록
- 사용자 리뷰는 실 이용자 기반 UGC

### 📊 데이터 수집 방법 요약

| 서비스 | 1차 데이터 소스 | 리뷰/별점 | 사진 |
|--------|----------------|----------|------|
| 데이트팝 | 자체 큐레이션 + 제휴 | 자체 UGC | 자체 촬영 + 제휴처 |
| 마이리얼트립 | 호스트 등록 + API | 자체 UGC | 호스트/사용자 |
| 다이닝코드 | 블로그 빅데이터 분석 | 외부 크롤링 + 분석 | 외부 크롤링 |
| 망고플레이트 | 자체 UGC | 자체 UGC | 사용자 업로드 |
| 캐치테이블 | B2B 식당 등록 | 실 이용자 UGC | 식당 제공 |

## 5. 바이브랩스 (VibeLabs) — 2026년 기준

### ⚠️ 조사 결과

"바이브랩스(VibeLabs)"에 대해 공식 웹사이트(vibelabs.ai, vibelabs.co 등)와 주요 검색엔진에서 확인한 결과, **2026년 2월 현재 널리 알려진 공식 프로그램으로 확인되지 않았습니다.**

가능성 있는 해석들:

#### 가능성 1: 네이버 "바이브(VIBE)" 관련 프로그램
- 네이버의 AI/음악 플랫폼 "바이브"와 관련된 개발자 프로그램
- 네이버 클라우드 스타트업 지원 프로그램의 일부일 수 있음

#### 가능성 2: 스타트업 액셀러레이터/인큐베이터
- 한국 또는 글로벌 스타트업 생태계의 신생 프로그램
- AI/XR 분야의 특화 액셀러레이터

#### 가능성 3: "바이브 코딩(Vibe Coding)" 관련
- 2025년부터 유행한 "바이브 코딩" 트렌드와 관련된 커뮤니티/랩
- AI를 활용한 코딩 생태계 지원 프로그램

## 🎯 종합 추천: 데이트코스 AI 서비스 데이터 아키텍처

```
┌─────────────────────────────────────────────────┐
│              데이트코스 AI 서비스                   │
├─────────────────────────────────────────────────┤
│                                                 │
│  [장소 기본정보]          [리뷰/평점/사진]          │
│  ├─ 카카오 로컬 API       ├─ Google Places API    │
│  ├─ TourAPI (관광공사)    ├─ 네이버 블로그 API     │
│  ├─ 공공데이터 (지자체)    │  + AI 감성분석        │
│  └─ 소상공인 상권정보      └─ 자체 UGC 시스템      │
│                                                 │
│  [AI 엔진]                                       │
│  ├─ 장소 데이터 통합/정규화 (Entity Resolution)    │
│  ├─ 블로그 리뷰 감성분석 & 요약                    │
│  ├─ 데이트코스 추천 알고리즘                       │
│  └─ 개인화 (취향, 예산, 위치 기반)                 │
│                                                 │
│  [예상 월 비용 (MVP)]                             │
│  ├─ 카카오 API: 무료 (월 300만건)                  │
│  ├─ 네이버 API: 무료 (일 25,000건)                │
│  ├─ TourAPI: 무료                                │
│  ├─ Google Places: $0~200/월 (무료크레딧)         │
│  └─ 합계: $0~50/월 (MVP 기준)                    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 핵심 전략
1. **카카오 + 네이버 + TourAPI로 장소 기본 데이터 확보** (무료)
2. **Google Places API로 리뷰/별점/사진 확보** (무료크레딧 활용)
3. **네이버 블로그 API + LLM으로 한국어 리뷰 분석** (합법적)
4. **자체 UGC 시스템 조기 구축**으로 데이터 자산 확보
5. **Entity Resolution**: 같은 장소를 여러 API에서 가져오므로, 이름+주소+좌표 기반 통합 필수
