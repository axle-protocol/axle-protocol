# Mac mini 팜 — 대량 수익화 1페이지 (초안)

작성일: 2026-02-08

## 한 줄
**Mac mini 팜은 “macOS/GUI가 꼭 필요한 반복 업무”를 병렬 처리할 때만 돈이 된다.**

---

## 언제 실용적인가 (YES)
- iOS/mac 앱 빌드/테스트처럼 **macOS가 필수**
- API가 없어서 결국 사람이 하듯이 해야 하는 **GUI 자동화**
- “항상 켜진 세션(브라우저/앱 로그인)”을 안정적으로 유지해야 함

## 언제 별로인가 (NO)
- 그냥 웹서버/API 호스팅(리눅스 클라우드가 더 싸고 쉬움)
- 개인 카톡/메일 계정 대행(신뢰/법무/2FA/클레임 리스크 큼)

---

## 돈 되는 패키지 3종

### 패키지 A) iOS/mac 빌드·테스트 팜 (B2B, 정석)
**고객:** 앱 개발팀/에이전시

**제공:**
- PR/릴리즈 빌드 자동화, 테스트 리포트
- 코드서명/아카이빙/배포 자동화
- 실패 시 알림 + 재시도

**왜 팜이 필요:** macOS 필수 + 병렬 빌드로 개발 속도 직결

**과금:**
- 월 구독(좌석/동시 빌드 수 기준) + 추가 크레딧(빌드 분/릴리즈 건)

**리스크:** 낮음(업무/개발 영역)

---

### 패키지 B) GUI 자동화 운영 대행 (B2B, 고마진)
**고객:** 쇼핑몰/마케터/운영팀/1인 사업자

**제공:**
- “API 없는 서비스”를 사람처럼 조작하는 자동화
- 예: 리포트 다운로드, 상품 등록, 재고 체크, 알림/정리
- 24/7 관제 + 장애 대응 + 로그/영수증

**왜 팜이 필요:** 24/7 + 여러 계정/여러 작업 병렬 + 세션 유지

**과금:**
- 셋업비 + 월 운영비 + 작업량(계정/워크플로 수) 과금

**리스크:** 중간(대상 사이트 정책/변경 대응 필요)

---

### 패키지 C) Agent Runtime 호스팅 (인프라 + 운영)
**고객:** 에이전트/자동화 쓰는 개인/팀

**제공:**
- OpenClaw 같은 에이전트를 24/7 구동
- 관제(비용캡/킬스위치/승인)

**왜 팜이 필요:** 세션 유지/GUI 자동화/멀티 워크플로 병렬

**과금:**
- 노드 1대당 월 과금 + 옵션(보안/승인/로그)

**리스크:** 중간(유스케이스가 약하면 ‘호스팅’으로 끝남)

---

## SOME(썸/카톡)와의 관계
- MVP: 유저 기기 로컬 실행이 가장 안전/쉬움
- 팜은 나중에 “Always-on Managed 옵션”으로 업셀 가능하지만,
  대량 수익화는 A/B 같은 B2B 패키지가 더 현실적.

---

## 결론
- **대량 수익화 우선순위:** B(운영 대행) ≈ A(빌드팜) > C(에이전트 호스팅) >>> 개인 계정 대행
- 팜은 ‘하드웨어’가 아니라 **운영/관제/장애 대응**을 팔아야 돈이 됨.
