# SOME(카톡 썸 답장 코치) — 법/정책 리스크 & 안전 운영 가이드 (KR)

작성일: 2026-02-09

> 목적: “법적으로 안전한 제품”을 보장하는 문서가 아니라,
> **리스크를 줄이는 운영 원칙/제품 설계 원칙**을 명확히 한다.

## 1) 큰 그림: 형사 리스크보다 ‘약관/정책/개인정보’ 리스크가 더 크다
- 한국에서 메신저 자동화는 보통 형사 처벌 이슈보다
  - **플랫폼 약관/운영정책 위반**(계정 정지/제한)
  - **개인정보 처리**(저장/전송/제3자 제공)
  - **민감 대화(성/자해/괴롭힘)** 대응
  쪽이 실제 운영 리스크.

참고 법령(원문):
- 개인정보보호법(국가법령정보센터): https://www.law.go.kr/lsEfInfoP.do?lsiSeq=195062
- 정보통신망법(국가법령정보센터): https://www.law.go.kr/LSW/lsInfoP.do?lsId=000030&ancYnChk=0

참고 정책(원문):
- 카카오톡 운영정책: https://talksafety.kakao.com/policy

## 2) 제품 포지션(리스크 최소화)
### ✅ 추천 포지션
- **‘자동 전송 봇’이 아니라 ‘답장 코치’**
- 기본 동작: 사용자가 메시지를 보고 → SOME가 **후보 3개+추천 1개** 생성 → 사용자가 선택/승인

### ⚠️ Autopilot Window는 ‘옵션’
- 자동 전송은 약관/정책 리스크가 커질 수 있어
- MVP는 반드시:
  - 시간창(최대 120분)
  - 화이트리스트(특정 대화방/상대만)
  - STOP/PAUSE 즉시 반영
  - 감사로그/증적
  - 금칙어/경계필터
  를 전제로 한다.

## 3) 개인정보/프라이버시 기본 원칙
- 기본값: **대화 원문 서버 저장 0** (로컬 처리)
- 텔레그램 관제에는 원문 대신:
  - 요약/마스킹
  - 필요 시 사용자가 “원문 전송”을 수동 승인
- 저장이 필요한 경우(운영/디버깅):
  - 최소 기간
  - 사용자가 삭제 가능
  - 민감정보(주소/계좌/링크 등) 자동 마스킹

## 4) 위험 대화(경계필터) 정책
- 아래 상황은 `shouldBlock=true` 기본:
  - 금전 요구/송금/계좌/현금 요구
  - 주소/위치/개인정보 과도 요구
  - 성적 동의/강요/불법 촬영/미성년 관련
  - 불법 행위(마약/도박/사기)
  - 괴롭힘/혐오/협박
  - 자해/자살 관련
- 차단 시 동작:
  - 자동 전송 금지
  - 사용자에게 ‘왜 차단했는지’와 안전한 대체 표현을 제공

## 5) 운영 가이드(유저에게 주는 사용법)
- “SOME는 내 대신 보내는 게 아니라, 내가 보낼 말을 코치해준다”
- Autopilot은:
  - 반드시 시간창 설정
  - 상대(대화방) 화이트리스트 설정
  - 술/감정 격한 상황에서는 OFF 권장

## 6) 면책(초안)
- SOME는 연애/대화 코칭 도구이며, 사용자 메시지 전송의 최종 책임은 사용자에게 있음
- SOME 사용으로 발생하는 계정 제재/정지 등은 플랫폼 정책에 좌우될 수 있음
- SOME는 사용자의 개인정보를 기본적으로 서버에 저장하지 않으며(로컬 기본), 사용자가 선택적으로 공유할 수 있음
