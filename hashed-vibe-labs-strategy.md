## 1. 현재 계획의 치명적 문제점

### 🚨 KILL LIST: 이건 망한다

**문제 #1: 마켓플레이스 = 3일에 만들 수 없다**

마켓플레이스는 **닭과 달걀 문제**의 극치다. 공급(AI 에이전트)도 필요하고, 수요(사용자)도 필요하다. 둘 다 없으면 둘 다 안 온다. Fiverr는 이 문제를 해결하는 데 **3년** 걸렸다. 3일이 아니라.

마켓플레이스 MVP에 최소한으로 필요한 것:
- 에이전트 등록/검색 시스템
- 결제 시스템 (PG 연동)
- 에이전트 실행 인프라
- 리뷰/평점 시스템
- 사용자 인증
- 에이전트 제작자용 대시보드

이걸 3일에? **진짜 바이브 코딩의 달인이어도 불가능하다.** 껍데기는 만들 수 있어도 실제로 돈이 오가는 결제 + 에이전트 실행까지 안정적으로? No.

**문제 #2: 47jobs가 이미 4개월 전에 런칭했다**

47jobs.com은 2025년 9월에 HN에서 런칭했다. "AI 에이전트를 프리랜서처럼 고용하는 마켓플레이스" — **정확히 당신이 만들려는 것**. 영어 시장이지만, "한국판 47jobs"라고 설명하는 순간 심사위원은 이렇게 생각한다:

> "아, 카피캣이네."

**문제 #3: "AI 앱스토어" 컨셉은 이미 레드오션**

- 몰트북: 153만 에이전트 등록 (당신의 마켓플레이스가 Day 1에 몇 개의 에이전트를 가질 수 있나? 10개? 50개?)
- 봇마당: 업스테이지 CEO가 **직접** 만들었다. 한국 AI 생태계 최고 인사이더.
- 머슴닷컴: 이미 바이럴 치고 있는 한국판 AI 커뮤니티
- Kore.ai: 엔터프라이즈급 AI 에이전트 마켓플레이스

**문제 #4: 마켓플레이스는 "트랙션"을 보여주기 가장 어렵다**

해시드가 원하는 건:
> "배포 이력과 개선 속도 등 **실제 운영 데이터**를 기반으로 성과를 점검"

마켓플레이스의 운영 데이터 = GMV (총 거래액). 13일 마케팅으로 의미 있는 GMV를 만들 수 있나? **₩5,000,000 자본으로?** 

구글/네이버 광고 CPA가 한국에서 최소 ₩5,000~₩15,000이다. 500만원이면 최대 333~1,000명의 유저를 데려올 수 있다. 그 중 마켓플레이스에서 실제 결제까지 하는 비율? 낙관적으로 5%. **16~50건의 거래.** 이걸로 심사위원을 설득?

**문제 #5: Product A + Product B = 집중력 분산**

솔로 파운더가 3일 안에 **두 개의 프로덕트**(마켓플레이스 + 카톡 연애봇)를 만든다? 이건 자살 미션이다. 하나를 제대로 못 만든다.

### 냉정한 결론

현재 계획대로 가면:
- Day 3: 반쯤 완성된 마켓플레이스 + 버그 많은 카톡 봇
- Day 5-16: "왜 아무도 안 쓰지?" 하면서 마케팅에 돈 태움
- Day 16: 유의미한 트랙션 없이 지원서 제출
- 결과: **탈락**

## 2. 경쟁사 차별화 전략

### 47jobs와의 차별화

| 47jobs | 당신이 가야 할 방향 |
|--------|-------------------|
| 영어 시장, 글로벌 | **한국어 특화** (한국 비즈니스 문맥 이해) |
| "에이전트를 고용한다" (수동적) | **에이전트가 먼저 제안한다** (능동적) |
| 범용 작업 (코딩, 콘텐츠, 분석) | **한국 특수 니즈** (세금신고, 정부24, 네이버 블로그 관리 등) |
| 웹 기반 | **카카오톡 네이티브** |
| B2C 중심 | **소상공인/1인기업 타겟** |

**핵심:** 47jobs는 "AI가 일을 한다"에 집중. 당신은 **"한국인의 일상 문제를 AI가 즉시 해결한다"**에 집중해야 한다.

### 몰트북/봇마당과의 차별화

이건 사실 경쟁이 아니다. **완전히 다른 카테고리다.**

- 몰트북/봇마당 = **AI들끼리 노는 SNS** (인간은 구경)
- 당신의 서비스 = **인간이 AI를 써서 문제를 해결하는 곳**

몰트북은 "구경거리"이고, 당신의 서비스는 "도구"여야 한다. 몰트북의 바이럴을 타되, 그 유저들을 당신의 실용적 서비스로 전환하는 게 전략이다.

## 3. 카톡 연애봇 판단

### 시장은 진짜다 — 돈이 된다

한국 캐릭터 챗 시장 데이터 (2025년 기준):

| 서비스 | 월 매출 추정 |
|--------|------------|
| 뤼튼 (캐릭터챗) | ₩20억 |
| 제타 (스캐터랩) | ₩20억 |
| 러비더비 | ₩10억 |
| 베이비챗 | ₩10억 |
| 로판AI | ₩5억 |
| 케이브덕 | ₩5억 |
| 기타 | ₩10억+ |
| **합계** | **₩80억+/월 = ₩960억/년** |

하드코어 유저 포함 시 **연 ₩2,400억 시장**으로 추정. 웹툰 시장이 2017년에 ₩3,800억이었다는 걸 생각하면, **캐릭터챗은 이미 검증된 거대 시장**이다.

### 🚫 그러나 카톡 연애봇은 안 된다

**기술적 킬러:** 카카오톡 채널 API는 **5초 타임아웃** 제한이 있다. AI 생성 응답이 5초를 넘기면 타임아웃. Gemini API 평균 응답 시간이 2-8초인데, 프롬프트가 복잡한 연애 시뮬레이션은 **거의 확실히 5초를 넘긴다.**

콜백 API로 우회할 수 있지만:
- UX가 구려진다 (응답이 분리됨)
- 대화 흐름이 끊긴다
- 연애 챗봇에서 **응답 지연 = 몰입 파괴 = 유저 이탈**

**경쟁 킬러:** 뤼튼(투자금 480억), 스캐터랩/제타, 러비더비 등이 이미 수백억 투자를 받고 전문 팀으로 운영 중이다. 솔로 파운더가 ₩500만으로 이 시장에서 싸운다? **이건 총 들고 탱크 앞에 서는 것**이다.

**법적 리스크:** 스캐터랩의 이루다 사태(개인정보 이슈)를 기억하라. AI 연애 서비스는 한국에서 **규제 리스크**가 높다. 개인정보보호위원회가 바로 어제(2/2) AI 프라이버시 민·관 협의회를 발족했다.

### 결론: 카톡 연애봇은 킬러 프로덕트가 아니다

시장은 맞지만, **전장이 잘못됐다.** 카톡 플랫폼의 기술 제약 + 기존 플레이어의 자본력 + 규제 리스크 = 실패 확률 극히 높음.

## 4. 13일 안에 트랙션 보여줄 MVP

### 🎯 제안: "한국 소상공인을 위한 AI 직원" (단일 프로덕트)

마켓플레이스를 접어라. **하나의 킬러 에이전트**를 만들어라.

**왜 소상공인인가?**

- 한국 소상공인 수: **약 700만 명** (자영업자 포함)
- 이들의 공통 고통: 마케팅을 못한다 (네이버 블로그, 인스타, 배달앱 리뷰 관리)
- 월 마케팅 예산: ₩50,000~₩300,000 (충분히 지불 의향)
- **아무도 이 시장을 AI 에이전트로 제대로 풀고 있지 않다**

### MVP 스펙: "AI 사장님" (가칭)

**핵심 기능 딱 1개:**

> "사장님, 가게 이름이랑 메뉴 알려주세요. 네이버 블로그 글 5개 바로 써드릴게요."

그게 끝. 이게 전부.

**기술 스택 (현재 보유 기술로 충분):**

```
Next.js (프론트) + Supabase (DB/Auth) + Gemini API (글 생성) + Vercel (배포)
카톡 채널 → 이건 5초 제한 때문에 메인이 아니라 "알림" 채널로만 활용
메인 UX → 웹앱 (모바일 최적화)
```

**3일 빌드 타임라인:**

| 일자 | 할 일 | 산출물 |
|------|-------|--------|
| Day 1 (2/3) | 랜딩페이지 + 핵심 플로우 | 업종/가게명 입력 → AI가 블로그 글 5개 생성 |
| Day 2 (2/4) | 결제 연동 + 사용자 대시보드 | 토스페이먼츠 간편결제 + 생성된 글 관리 |
| Day 3 (2/5) | 카톡 알림 + 온보딩 | 생성 완료 알림 + 첫 사용자 유입 시작 |

**13일 마케팅 타임라인:**

| 기간 | 전략 | 예상 지표 |
|------|------|----------|
| Day 4-6 | 네이버 소상공인 카페 직접 홍보 (무료) | 가입 100명 |
| Day 7-9 | 배민/요기요 리뷰 많은 가게에 카톡 DM | 가입 200명 |
| Day 10-12 | 첫 주 데이터로 개선 + 유료 전환 | 유료 30명 |
| Day 13-16 | 입소문 + 리퍼럴 (친구 소개 시 무료 1회) | 유료 80명+ |

### 왜 이게 트랙션을 만드는가

1. **즉각적 가치**: 블로그 글 1개 외주 비용 = ₩30,000~₩100,000. AI로 ₩5,000에? 즉시 체감.
2. **측정 가능**: 생성된 글 수, 유료 전환율, 리텐션 — 모두 **숫자로 보여줄 수 있다**.
3. **입소문**: 소상공인끼리 **카톡 단톡방/지역 카페에서 공유**한다. 바이럴 구조 내장.
4. **반복 구매**: 블로그 글은 매주/매월 필요하다. **리커링 수요**.

### 대안 킬러 프로덕트 (고려할 것)

연애봇 대신 이런 것들이 더 낫다:

| 아이디어 | 타겟 | 트랙션 가능성 | 수익화 |
|---------|------|-------------|--------|
| **네이버 블로그 AI 대필** | 소상공인 700만 | ⭐⭐⭐⭐⭐ | 즉시 |
| **부동산 매물 설명 AI 생성** | 공인중개사 12만 | ⭐⭐⭐⭐ | 즉시 |
| **쿠팡/스마트스토어 상품 상세 AI** | 온라인 셀러 | ⭐⭐⭐⭐ | 즉시 |
| **세무 서류 AI 정리** | 소상공인/프리랜서 | ⭐⭐⭐ | 높음 |
| **학부모용 AI 학습 코치** | 학부모 | ⭐⭐⭐ | 중간 |

**최고 추천: 네이버 블로그 AI 대필** — 가장 빠르게 만들 수 있고, 가장 쉽게 돈 받을 수 있고, 가장 바이럴 가능성이 높다.

## 5. 한국 시장 인사이트

### 한국인이 실제로 AI에 돈 쓰는 곳

1. **캐릭터챗/연애 시뮬레이션**: 월 ₩80억+ (입증됨)
2. **AI 글쓰기 (뤼튼, 노션AI)**: 급성장 중
3. **AI 이미지 생성 (미드저니, 달리)**: 크리에이터 사이 대중화
4. **AI 코딩 (커서, GitHub Copilot)**: 개발자 필수 도구화

### 한국 특수 니즈 — 아무도 안 풀고 있는 것

1. **네이버 SEO 최적화 AI**: 네이버 블로그가 여전히 한국 검색의 핵심. 구글 SEO 도구는 많지만 네이버 특화는 없음.
2. **카카오톡 비즈니스 자동화**: 카톡 채널 운영하는 소상공인은 많은데, 메시지 자동 응답이 구림.
3. **정부 보조금 신청 AI**: 소상공인/스타트업 보조금이 수백 종. 어디에 지원 가능한지 찾는 것만 반나절.
4. **한국어 특화 프롬프트**: ChatGPT를 영어로 써야 제대로 되는데, 한국어 프롬프트 최적화 도구가 없음.

### 한국 유저 행동 특성

- **네이버 카페 = 바이럴의 핵심**: 관심사별 카페가 마케팅 채널. 소상공인 관련 카페 (자영업자 커뮤니티, 배달의민족 사장님 카페 등) 가입자 수만 합쳐도 수십만.
- **"무료 체험 → 카톡 공유 → 유료 전환"**: 한국인의 전형적 구매 패턴.
- **가격 민감**: 월 ₩9,900~₩29,900이 골든 프라이스. ₩50,000 넘으면 급격히 전환율 하락.
- **레퍼런스 중시**: "옆 가게 사장님도 쓴다"가 최고의 마케팅.

## 6. 수익모델 현실 점검

### 현재 계획: 15-20% 거래 수수료 + 프리미엄 구독

**문제:**

- 거래 수수료 15-20%는 **에이전트 제작자가 수용하지 않을 가격**이다. 앱스토어도 30%에서 15%로 내리는 추세. AI 에이전트 마켓에서 유통 파워가 없는 신생 플랫폼이 15-20% 떼간다? 에이전트 제작자가 안 온다.
- 구독 모델은 "마켓플레이스에서 뭘 구독하는데?"라는 근본 질문에 답을 못함.

### 제안 수익모델 (블로그 AI 대필 기준)

```
무료 체험:  블로그 글 3개 무료 생성
유료 전환:  
  - 기본 (₩9,900/월): 월 20개 글 생성
  - 프로 (₩29,900/월): 무제한 + 네이버 SEO 최적화 + 이미지 자동 생성
  - 비즈 (₩99,000/월): 프로 + 인스타/카톡 채널 콘텐츠 자동 생성
```

**단가 현실 점검:**

- Gemini API 비용: 글 1개 생성 ≈ ₩50~₩200 (프롬프트 복잡도에 따라)
- 기본 플랜 20개 글: API 비용 ₩1,000~₩4,000 → **마진 ₩6,000~₩9,000 (60~90%)**
- 프로 플랜: 월 평균 50개 글 가정 → API 비용 ₩10,000 → **마진 ₩20,000 (67%)**

**13일 후 수익 시뮬레이션:**

| 지표 | 비관적 | 현실적 | 낙관적 |

비관적이어도 **"실제 결제하는 유저가 있다"**는 것 자체가 해시드에게 강력한 시그널이다. 마켓플레이스의 GMV ₩0보다 100배 낫다.

## 7. 해시드 심사위원을 설득하는 법

### 해시드가 보는 평가 기준 (공식 발표)

ZDNet 기사에서 직접 인용:

1. ✅ **빠른 매출 신장 중심의 AI 네이티브 사고방식**
2. ✅ **배포 빈도 및 속도**
3. ✅ **반복 개선 속도와 변화의 질**
4. ✅ **작업 분해 및 AI 위임 방식**
5. ✅ **오류·실패 복구 방식**
6. ✅ **실행 루프의 안정성**

### 각 기준별 공략법

**1. 빠른 매출 신장**
→ Day 1부터 수익화 가능한 프로덕트. 무료→유료 전환 그래프를 보여줘라. 마켓플레이스의 "언젠가 수익화할게요"가 아니라, **"지금 결제가 들어오고 있다"**.

**2. 배포 빈도 및 속도**
→ GitHub commit history를 보여줘라. 3일 빌드 동안 **하루 10회 이상 배포**. Vercel의 배포 로그가 증거. 이건 바이브 코딩의 본질 — **코드를 빠르게 밀어낸다**.

**3. 반복 개선 속도**
→ 유저 피드백 → 수정 → 배포 사이클을 **시간 단위**로 보여줘라. "2/10 14:00 유저 A가 '블로그 글이 너무 딱딱하다' 피드백 → 14:30 프롬프트 수정 → 15:00 배포 → 15:30 유저 A '훨씬 낫다' 응답" 이런 로그.

**4. 작업 분해 및 AI 위임**
→ **당신이 AI를 얼마나 깊이 활용하는지** 보여줘라:
  - 코드 작성: Cursor/Claude로 90% 자동 생성
  - 프로덕트 기능: Gemini API가 핵심 가치 전달
  - 마케팅 문구: AI 생성
  - 고객 지원: AI 자동 응답
  - **"나는 의사결정만 한다. 나머지는 AI가 한다"**

**5. 오류·실패 복구**
→ 의도적으로 빠르게 실패하고 복구한 기록을 남겨라. "2/4 API 비용이 예상의 3배 → 프롬프트 최적화로 60% 절감 → 2시간 만에 해결" 같은 로그.

**6. 실행 루프 안정성**
→ 모니터링 대시보드. Vercel Analytics + Supabase 실시간 데이터. "서비스가 24시간 안정적으로 돌아간다."

### 💀 심사위원의 머릿속

김서준 해시드 대표의 인용:

> "무엇을 만들 가치가 있는지 판단하고 그 선택을 글로벌 시장 끝까지 밀어붙이는 사람을 발견하고 지원하는 과정"

> "실행은 누구에게나 열려 있지만, **방향을 정하고 품질을 통제하는 안목과 책임**은 여전히 창업자의 몫"

**번역:** "바이브 코딩으로 아무거나 빨리 만드는 사람"이 아니라, **"올바른 것을 빨리 만드는 사람"**을 찾고 있다.
