# 에이전트마켓 딥 프로덕트 분석 (2026-02-05)

## 🔴 핵심 발견: "AI가 입찰만 하고 실제 일은 안 한다"

현재 최대 gap은 **태스크 실행 엔진**이 없다는 것.
유저가 태스크를 올리면 → AI가 입찰 → 유저가 선택 → **그 다음이 없음**

AI가 실제로 번역/카피/SEO/코드리뷰를 수행하고 결과물을 납품하는 시스템이 빠져있음.
이게 없으면 마켓플레이스가 아니라 "입찰 시뮬레이터"임.

---

## 📋 유저 여정 완전 분석 (빈 구간 식별)

```
회원가입 → ✅ 있음
크레딧 충전 → ✅ 있음 (토스페이먼츠)
태스크 올리기 → ✅ 있음
에이전트 입찰 → ✅ 있음 (Auto-bid)
입찰 선택 → ⚠️ UI 미흡
→ AI 작업 수행 → ❌ 없음 (핵심 gap)
→ 결과물 납품 → ❌ 없음
→ 결과물 확인 → ❌ 없음
→ 승인/수정요청 → ❌ 없음
→ 크레딧 차감 → ❌ 있음 (함수만, UI 없음)
→ 리뷰/평점 → ❌ 없음
→ 재주문 → ❌ 없음
```

## 🔴 Critical: 제품이 작동하지 않는 것들

### 1. 태스크 실행 엔진 (Task Execution Engine)
- AI 에이전트가 실제로 작업을 수행하는 백엔드
- Groq LLM으로 번역/카피/리서치/코드리뷰 실제 처리
- 결과물을 텍스트/마크다운으로 생성 → DB 저장 → 유저에게 표시

### 2. 유저 대시보드 (/dashboard)
- 현재 404 에러
- 내 태스크 목록 (진행중/완료/대기)
- 내 크레딧 잔액 + 사용 내역
- 알림 센터

### 3. 크레딧 잔액 표시
- 네비바나 헤더에 AM$ 잔액 표시 (토스 스타일)
- 로그인 시 항상 보이게

### 4. 납품물 보기 UI
- 태스크 상세 페이지에서 에이전트가 납품한 결과물 확인
- 마크다운 렌더링, 복사 버튼
- 승인/수정요청 버튼

### 5. 결제 흐름 완성
- 에이전트 선택 → 에스크로 차감 → 작업 완료 → 승인 → 지급
- 현재는 충전만 되고 사용하는 플로우가 없음

## 🟡 Important: 실제 유저 받기 전 필요

### 6. 알림 시스템
- 최소 인앱 알림 (대시보드에 뱃지)
- 이상적: 카카오톡 알림 or 이메일

### 7. 리뷰/평점
- 태스크 완료 후 1~5별 평점 + 한줄 리뷰
- 에이전트 프로필에 평점 표시

### 8. 전자상거래 법적 표시
- 사업자정보 푸터에 표시 (사업자번호: 726-02-02763)
- 통신판매업 신고 필요 여부 확인
- 이용약관/개인정보처리방침 실제 내용

### 9. 카카오 로그인 완성
- 한국 유저의 70%+ 기대하는 로그인 방식
- 네이버 로그인도 추후 추가

### 10. 파일 업/다운로드
- 태스크에 참고 파일 첨부
- 납품물로 파일 다운로드
