# 한국 UI/UX + 비교광고 법률 리서치 (2026-02-05)

## 🎨 한국 인기 앱 UI/UX 핵심 패턴

### 토스 (Toss) — 금융 앱 국민 1위
1. **한 페이지 한 태스크** — 단계마다 페이지 분리, 정보 복잡성 최소화
2. **심플 정보 계층** — 핵심 수치만 보여주고 세부사항은 하위 페이지
3. **한글 뱃지** — 'NEW' 대신 '새로 나온', '인기' 등 한글 사용
4. **로딩 피드백** — 0.4초 이상 대기 시 반드시 시각적 애니메이션
5. **광고 = 콘텐츠** — 광고도 일반 리스트와 동일 UI, 부담감 제거
6. **신뢰 소구** — "이렇게 쉬운 송금이 안전한 이유" 별도 섹션
7. **금액 + 보상** — 광고 3초 보면 포인트 지급, 불쾌감 최소화

### 배달의민족 — UX 라이팅 최강
1. **사용자 맥락 우선** — "3명 선택 완료" → "2명 남음" (중요 정보 우선)
2. **마음의 준비** — 복잡한 절차 전 "2분 만에 끝내기" 시간 예고
3. **법적 정보 최소화** — 핵심만 간단히, 법적 근거는 접기
4. **친근한 톤** — 배민체, 유머, 한글 중심

### 네이버 — 포털/커머스
1. **카드형 UI** — 뉴스, 상품, 검색결과 모두 카드 모듈
2. **검색 중심 UX** — 메인 CTA가 항상 검색
3. **탭 기반 네비게이션** — 하단 5탭 구조
4. **그린/화이트** — 신뢰감 컬러

### 숨고/크몽 — 프리랜서 마켓
1. **카테고리 그리드** — 서비스를 아이콘+텍스트 그리드로 탐색
2. **견적/입찰 플로우** — 의뢰→견적→비교→선택→결제
3. **리뷰/평점 중심** — 별점 + 후기가 신뢰 핵심
4. **가격 투명성** — "~부터" 시작가 표시
5. **프로필 = 포트폴리오** — 전문가 프로필에 작업물 포함

## 🏗️ 에이전트마켓에 적용할 한국식 UX 원칙

1. **원스텝 원태스크** — 의뢰 등록을 단계별 UI로 (토스 스타일)
2. **한글 중심 UX 라이팅** — "시작하기", "견적 받기", "완료!" 
3. **시간 예고** — "AI가 약 30초 안에 입찰합니다" (배민 스타일)
4. **카드형 서비스 카테고리** — 아이콘 + 서비스명 + 시작가 (숨고 스타일)
5. **신뢰 섹션** — "AI가 정말 일을 할 수 있나요?" FAQ (토스 스타일)
6. **에스크로 안심 결제** — "만족할 때만 결제" 강조 (네이버 스마트스토어 참고)

## ⚖️ 비교광고 법률 분석

### 관련 법률
- **표시·광고의 공정화에 관한 법률** (표시광고법) 제3조
- **부정경쟁방지 및 영업비밀보호에 관한 법률**

### 비교광고 4가지 유형 (위법 기준)
1. **거짓·과장** — 사실과 다르거나 과장 → 위법
2. **기만** — 오인하게 만듦 → 위법  
3. **부당한 비교** — 기준 없이 우량 주장 → 위법
4. **비방** — 경쟁사 비하 목적 → 위법

### 합법적 비교광고 조건 (공정위 심사지침)
1. ✅ 비교 기준을 명확히 제시
2. ✅ 객관적 사실에 근거
3. ✅ 경쟁사 정보를 사실대로 표시
4. ✅ 비방 목적이 아닌 정보 제공 목적
5. ✅ 소비자에게 유용한 정보

### ⚠️ 에이전트마켓 적용 결론
- **크몽/숨고 직접 명시 = 리스크 있음**
  - 가격 비교 시 객관적 근거 없으면 "부당 비교"
  - 소규모 스타트업이 대기업 상대 법적 분쟁 = 불리
  - Han 우려 타당

### ✅ 안전한 대안
1. **일반 용어 사용**: "기존 프리랜서 플랫폼", "일반 외주 마켓"
2. **카테고리 비교**: "인간 프리랜서 방식" vs "AI 에이전트 방식"
3. **간접 표현**: "기존 방식의 1/10 가격" (특정 업체 미지정)
4. **자사 강점만 강조**: "AI 에이전트가 5분 만에 완료"
5. **사실 근거 시**: 가격 비교표에 "시장 평균가" 사용 (특정 업체 X)
