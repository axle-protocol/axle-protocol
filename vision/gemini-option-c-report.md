## 핵심 컨셉

**"OpenClaw Node Network & The Agent Passport Protocol"**

> "AI 에이전트를 위한 Amazon Mechanical Turk"

---

## 문제 정의

1. 순수 AI 에이전트는 CAPTCHA 못 뚫음, SMS 인증 불가
2. 클라우드 에이전트는 IP 차단에 취약
3. 사용자는 어떤 에이전트가 믿을 만한지 모름 (신원/평판 부재)

---

## 솔루션

### 1. 분산형 실행 노드 (DePIN)
- 전 세계 사용자 PC의 OpenClaw = **노드**
- 각자 거주지 IP + 브라우저 지문으로 작업 수행
- 예: 한국 노드 → 네이버 예약, 미국 노드 → 아마존 구매

### 2. 온체인 신원 증명 (ERC-8004)
- 모든 OpenClaw 노드가 자신의 신원/능력을 온체인 등록
- 에이전트 메타데이터(기능, 요금, 엔드포인트)를 **NFT 형태**로 발급

### 3. 작업 증명 (Proof of Task)
- 에이전트가 작업 완료 → 결과(스크린샷, TX 해시) 암호화 증명
- 평판 점수(Reputation) 획득

## 기술 스택

| 구성 요소 | 기술 | 역할 |
|-----------|------|------|
| **신원 레지스트리** | ERC-8004 (EIP) | 에이전트 NFT 발급, P2P 발견 |
| **평판 시스템** | EAS (Ethereum Attestation Service) | 작업 성공 온체인 기록 |
| **검증 레이어** | TEE / zk-TLS (Notary) | 브라우징 결과 조작 방지 |
| **보안** | Docker 샌드박스 | 파일시스템 접근 제한 |

## 비즈니스 모델

- **Network Fee**: 작업당 트랜잭션 수수료
- **Permissionless**: 누구나 노드 참여 가능
- **토큰 이코노미**: 노드 운영자 보상

---

## 경쟁 우위 (Moat)

1. **네트워크 효과**: 노드 ↑ → 작업 종류 ↑ → 에이전트 유입 ↑
2. **평판 데이터**: 온체인 작업 기록 = 복제 불가능한 자산
3. **규제 우회**: P2P 구조로 중개자 책임 분산
